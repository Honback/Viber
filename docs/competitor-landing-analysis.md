# 경쟁/유사 플랫폼 랜딩페이지 구조 분석

## 1. Product Hunt

> "The best new products in tech."

### 랜딩 구조

```
┌─────────────────────────────────────────┐
│ Header: 로고 + 네비게이션 + 검색 + 로그인 │
├─────────────────────────────────────────┤
│ 오늘의 제품 리스트 (메인)                 │
│ - 순위 + 아이콘 + 이름 + 한줄소개         │
│ - 카테고리 태그                          │
│ - 업보트 수                              │
│ - 중간중간 Promoted(광고) 삽입            │
├─────────────────────────────────────────┤
│ 뉴스레터 구독 CTA                        │
├─────────────────────────────────────────┤
│ Yesterday's Top Products (Top 5)        │
├─────────────────────────────────────────┤
│ Last Week's Top Products (Top 5)        │
├─────────────────────────────────────────┤
│ Last Month's Top Products (Top 5)       │
└─────────────────────────────────────────┘
```

### 특징
- **Hero 없음** — 랜딩 = 곧 제품 피드. 설명 문구 최소화
- **리스트 중심** — 한 줄짜리 제품 카드가 세로로 나열
- **시간 기반 큐레이션** — 오늘/어제/지난주/지난달로 구분
- **업보트(투표) 시스템** — 순위 결정 핵심
- **Promoted 광고** — 리스트 사이에 자연스럽게 삽입 (수익 모델)
- **카테고리 태그** — 각 제품에 2~3개 태그

### Vibeollio 참고 포인트
- 랜딩에서 바로 제품을 보여주는 구조 (설명 < 콘텐츠)
- 시간 기반 섹션 분류 (오늘/이번주/이번달)
- Promoted 삽입형 수익 모델

## 2. BetaList

> "Discover tomorrow's startups, today."

### 랜딩 구조

```
┌─────────────────────────────────────────┐
│ Header: 카테고리 태그 가로 스크롤          │
├─────────────────────────────────────────┤
│ Hero (미니멀)                            │
│ - 제목: "Discover tomorrow's startups"   │
│ - 부제: 한줄                             │
│ - CTA: Submit startup / Sign in         │
├─────────────────────────────────────────┤
│ Trending Startups (3개 하이라이트)        │
│ - 아이콘 + 이름 + 한줄 설명              │
├─────────────────────────────────────────┤
│ 뉴스레터 구독 배너                       │
├─────────────────────────────────────────┤
│ 날짜별 제품 리스트                       │
│ Today → Yesterday → Wednesday...        │
│ - 아이콘(40px) + 이름 + 한줄 설명        │
│ - BOOSTED 태그 (유료 부스트)             │
│ - 하루 15~20개씩 나열                    │
├─────────────────────────────────────────┤
│ Load next page (페이지네이션)            │
└─────────────────────────────────────────┘
```

### 특징
- **Hero 극도로 심플** — 제목 + 부제 + 버튼 2개. 끝
- **날짜별 그룹핑** — Today, Yesterday, Wednesday... 역순 타임라인
- **리스트 밀도 높음** — 아이콘 작고, 한줄 설명, 여백 최소
- **BOOSTED** — 유료로 상단 노출 (수익 모델)
- **상단 카테고리 태그** — SaaS, AI Tools, Analytics 등 필터
- **Trending 섹션** — 3개만 크게 하이라이트

### Vibeollio 참고 포인트
- Hero를 짧게 끝내고 바로 콘텐츠로 진입
- 날짜별 타임라인 구조
- BOOSTED 유료 노출 모델
- Trending 하이라이트 (소수만 픽)

## 3. SaaSHub

> "Software Alternatives, Accelerators & Startups"

### 랜딩 구조

```
┌─────────────────────────────────────────┐
│ Header: 로고 + Register + Login          │
├─────────────────────────────────────────┤
│ Hero                                    │
│ - 제목 + "216,418 products and growing"  │
│ - 카테고리 태그 버튼 (CRM, Help Desk...) │
├─────────────────────────────────────────┤
│ SaaSHub Experts (게이미피케이션)          │
│ - 토너먼트 형식, 카운트다운 타이머         │
│ - "Top 5를 맞춰라" 참여형                │
├─────────────────────────────────────────┤
│ Top 5 Products (현재 라운드)             │
├─────────────────────────────────────────┤
│ Trending Products                       │
│ - 카드형 (이미지 + 이름 + 설명 + 평점)    │
├─────────────────────────────────────────┤
│ Featured Products                       │
├─────────────────────────────────────────┤
│ New Products                            │
├─────────────────────────────────────────┤
│ 카테고리별 탐색                          │
└─────────────────────────────────────────┘
```

### 특징
- **숫자 강조** — "216,418 products" 규모감
- **게이미피케이션** — Experts 토너먼트, Top 5 예측
- **카테고리 중심 탐색** — 상단에 주요 카테고리 버튼 배치
- **대안 찾기에 특화** — "~의 대안" 검색 구조
- **평점 시스템** — 별점으로 제품 평가

### Vibeollio 참고 포인트
- 게이미피케이션 요소 (참여 유도)
- 숫자로 플랫폼 규모감 전달
- 카테고리 버튼을 Hero 영역에 배치

## 4. AlternativeTo

> "Crowd-sourced app recommendations"

### 랜딩 구조

```
┌─────────────────────────────────────────┐
│ Header: 로고 + 검색바(중앙) + Sign up    │
├─────────────────────────────────────────┤
│ Hero                                    │
│ - 통계: "140,789 Apps / 1,987,306 Opinions" │
│ - 중앙 검색바 (핵심 인터랙션)             │
│ - 부제: "find better apps and services"  │
├─────────────────────────────────────────┤
│ 뉴스 섹션                               │
│ - 최신 테크 뉴스 카드 (이미지 + 제목)     │
├─────────────────────────────────────────┤
│ Trending Apps                           │
│ - 아이콘 + 이름 + 플랫폼 태그 + 좋아요 수 │
├─────────────────────────────────────────┤
│ New Apps                                │
├─────────────────────────────────────────┤
│ 카테고리 그리드                          │
└─────────────────────────────────────────┘
```

### 특징
- **검색 중심** — Hero의 핵심이 검색바 ("~의 대안을 찾아라")
- **크라우드소싱 강조** — 유저 의견 수를 대형 숫자로 표시
- **뉴스 콘텐츠** — 테크 뉴스로 트래픽 유입
- **플랫폼 태그** — Windows/Mac/Linux/Web 등 명확한 필터
- **심플한 카드** — 아이콘 + 이름 + 태그 + 추천 수

### Vibeollio 참고 포인트
- 검색을 Hero에 배치하는 방식
- 크라우드소싱 규모감 숫자
- 뉴스/콘텐츠로 SEO 트래픽 확보

## Vibeollio에 대한 시사점

### 1. 랜딩 = 콘텐츠
> 4개 사이트 모두 **기능 설명 섹션이 거의 없다**. 랜딩 자체가 제품 피드.

현재 Vibeollio: Hero → 카테고리 → How it works → 핵심 기능(6개) → 메이커 기능(6개) → 제품 리스트 → 통계 → 피드백 → CTA

**문제:** 기능 설명이 너무 많아서 실제 제품에 도달하기까지 스크롤이 길다.

### 2. 제안: 구조 단순화

```
[현재 - 10개 섹션]              →  [제안 - 5~6개 섹션]
1. Hero                            1. Hero (심플하게)
2. 카테고리 배너                     2. 트렌딩/추천 제품 (바로 콘텐츠)
3. How it works                     3. 신규 등록 제품
4. 핵심 기능 - 발견자                4. 피드백 요청 제품
5. 메이커 기능                      5. CTA
6. 신규 공개 (제품)                  6. Footer
7. 통계
8. 피드백 요청 (제품)
9. CTA
10. Footer
```

### 3. 핵심 원칙
- **Hero는 짧게** — BetaList처럼 제목 + 한줄 + CTA면 충분
- **설명 < 콘텐츠** — How it works, Feature Grid 같은 "설명" 섹션은 별도 About 페이지로
- **바로 제품을 보여줘라** — Product Hunt처럼 랜딩 = 피드
- **시간 기반 정렬** — 오늘/이번주 등 시간축으로 신선함 유지
