# Vibeollio - UI 섹션 정의서

```

  1. Header (고정)
  2. Hero

  4. How It Works
  5. Features
  6. Social Proof
  7. CTA (Final)
  8. Footer
```

---

## 1. Header

```
┌─────────────────────────────────────────────────────┐
│  [Vibeollio 로고]              [탐색] [등록하기] [로그인] │
└─────────────────────────────────────────────────────┘
```

- 스크롤 시 상단 고정 (sticky)
- 미니멀. 메뉴 3개 이하
- 모바일: 햄버거 메뉴

## 2. Hero

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│            당신이 만든 것을 세상에 보여주세요.           │
│                                                     │
│         아직 발견되지 않은 훌륭한 아이디어를 위한        │
```

| 요소 | 내용 |
|------|------|

| **강조 문구** | 아직 발견되지 않은 훌륭한 아이디어를 위한 홍보 플랫폼 |
| **CTA 버튼** | 지금 등록하기 (무료) |

- 배경: 클린한 단색 또는 미니멀 그라데이션
- 텍스트 외 장식 요소 없음

## 3. Problem-Solution

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  "좋은 제품인데 알릴 곳이 없어요"    → 무료로 등록하세요 │
│  "피드백을 받고 싶어요"             → 실시간으로 받으세요 │
│  "개발자가 아닌데 올려도 되나요?"    → 누구나 환영합니다  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| **형식** | 좌: 고민(회색) → 우: 해결(강조색) |
| **항목 수** | 3개 (간결하게) |

## 4. How It Works

```
┌─────────────────────────────────────────────────────┐
│                   3단계로 시작하세요                    │
│                                                     │
│    ①               ②                ③              │
│  URL 입력      설명 작성          피드 노출           │
│  홍보할 링크를   간단한 소개를      유저들에게           │
│  붙여넣으세요    적어주세요        바로 노출됩니다       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| **형식** | 3-step 가로 배치 (모바일: 세로) |
| **각 스텝** | 숫자 + 키워드 + 한줄 설명 |

## 5. Features

```
┌─────────────────────────────────────────────────────┐
│                  Vibeollio가 드리는 것                     │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │  노출     │  │  피드백   │  │  신뢰 배지 │          │
│  │  새로운   │  │  실시간   │  │  랜딩에    │          │
│  │  유저에게  │  │  솔직한   │  │  달 수 있는│          │
│  │  발견됩니다│  │  피드백   │  │  인증 마크 │          │
│  └──────────┘  └──────────┘  └──────────┘          │
│                                                     │
│  ┌──────────┐  ┌──────────┐                         │
│  │  유사 서비스│  │  포인트   │                         │
│  │  비교 탐색 │  │  피드백   │                         │
│  │          │  │  보상 체계 │                         │
│  └──────────┘  └──────────┘                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 카드 | 키워드 | 한줄 설명 |
|------|--------|-----------|
| 1 | **노출** | 새로운 유저에게 발견됩니다 |
| 2 | **피드백** | 솔직하고 실질적인 피드백을 실시간으로 |
| 3 | **신뢰 배지** | 내 랜딩페이지에 달 수 있는 인증 마크 |
| 4 | **유사 서비스** | 비슷한 도구와 비교하며 탐색 |
| 5 | **포인트** | 피드백 활동에 대한 보상 체계 |

- 그리드: 3열 (모바일: 1열)
- 각 카드: 아이콘 + 키워드 + 설명 한줄

## 6. Social Proof

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ← [제품카드] [제품카드] [제품카드] [제품카드] →       │
│                                                     │
│     "등록 3일 만에 피드백 20개를 받았어요"              │
│                                 — 제작자 A           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| **상단** | 등록된 제품 카드 캐러셀 (자동 슬라이드) |
| **하단** | 제작자 후기 인용문 1개 (로테이션) |

- MVP 전에는 목업 데이터 또는 "Coming Soon" 처리 가능

## 디자인 가이드라인

| 항목 | 방향 |
|------|------|
| **테마** | 라이트 기본 (다크 모드 지원 검토) |
| **컬러** | 화이트 배경 + 1개 포인트 컬러 + 그레이 텍스트 |
| **폰트** | 산세리프, 본문 16px 이상, 제목은 크고 굵게 |
| **여백** | 넉넉하게. 섹션 간 최소 80px |
| **애니메이션** | 스크롤 시 페이드인 정도. 과하지 않게 |
| **반응형** | 모바일 퍼스트. 그리드 → 단일 열 전환 |
