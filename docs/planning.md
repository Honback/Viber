# Vibeollio - 프로젝트 기획서

## 1. 개요

**Vibeollio**는 SaaS, 웹, 앱의 **제작자와 유저를 연결하는 홍보 플랫폼**이다.

> 만든 사람은 노출과 피드백을 얻고, 쓰는 사람은 새로운 도구를 발견한다.

---

## 2. 핵심 철학 (Heritage)

| # | 원칙 | 의미 |
|---|------|------|
| 1 | **등록 무료** | 진입장벽 제거. 누구나 자신의 제품을 올릴 수 있어야 한다 |
| 2 | **새로운 제품 발굴에 초점** | 크라우드펀딩처럼 인기 몇 개만 살아남는 구조가 아님. 롱테일 발굴 |
| 3 | **모든 아이디어는 훌륭하다** | 아직 발굴되지 못한 것뿐. 플랫폼이 발굴 기회를 제공 |
| 4 | **제작자가 얻는 것** | 노출 / 유저 확보 / 실질적 피드백 / 수익 기회 |

## 3. 유저 시나리오

### 3.1 제품 등록 플로우

```
[등록 버튼 클릭] → [정보 입력] → [등록 완료] → [피드 노출]
```

**입력 항목:**

| 필드 | 필수 여부 | 설명 |
|------|-----------|------|
| URL | 필수 | 홍보할 제품/서비스 링크 |
| 설명 | 필수 | 제품 소개 텍스트 |
| 이미지 | 선택 | 스크린샷, 로고 등 |

> **기획 노트:** 등록 항목은 최소화하되, 자동 크롤링(OG태그, 파비콘 등)으로 정보를 보완하는 방향 검토 필요

## 4. UI/UX 방향

- **레퍼런스:** [bkit.ai](https://www.bkit.ai)
- **원칙:** 심플. 불필요한 장식 배제, 콘텐츠 중심 레이아웃

### 4.1 bkit.ai UI 구조 분석

#### 페이지 구성 (싱글 페이지, 세로 스크롤)

```
┌─────────────────────────────────────────┐
│  Header: 로고(좌) + 언어 전환(우)        │  ← 미니멀 네비게이션
├─────────────────────────────────────────┤
│  Hero Section                           │
│  - 대형 타이틀 + 버전 태그              │
│  - 감성 카피 ("Don't worry.")           │
│  - CTA: 설치 코드 2줄 (복사 가능)       │
│  - 해시태그 3개                         │
├─────────────────────────────────────────┤
│  Problem-Solution Section               │
│  - 좌: 고민 목록 (회색 텍스트)          │
│  - 우: 해결 메시지 (강조 텍스트)        │
│  - 대화형 Q&A 포맷                      │
├─────────────────────────────────────────┤
│  Quick Start (2 Steps)                  │
│  - Step 1, Step 2 코드 블록             │
│  - "Done. Now start." 한줄 마무리       │
├─────────────────────────────────────────┤
│  Feature Grid (카드 레이아웃)           │
│  - 4~5개 핵심 기능 카드                 │
│  - 각 카드: 아이콘 + 키워드 + 한줄 설명 │
│  - NEW 배지로 신규 기능 강조            │
├─────────────────────────────────────────┤
│  Feature Deep Dive (탭/슬라이드)        │
│  - 4개 탭 전환 (01~04)                  │
│  - 각 탭: 설명 텍스트 + 터미널 시뮬레이션│
│  - 체크리스트(✓)로 기능 나열            │
├─────────────────────────────────────────┤
│  Social Proof (무한 스크롤 캐러셀)      │
│  - 유저 후기 카드 자동 슬라이드          │
│  - 익명 처리된 실명 ("Lee*jun")         │
│  - 한국어 원문 후기                     │
├─────────────────────────────────────────┤
│  Release Notes (타임라인)               │
│  - GitHub Stars 수 표시                 │
│  - 버전별 변경사항 아코디언              │
│  - "View all" → GitHub 링크             │
├─────────────────────────────────────────┤
│  Footer CTA                             │
│  - 대형 카피 + Get Started 버튼         │
└─────────────────────────────────────────┘
```

#### 디자인 특징

| 요소 | bkit.ai 적용 | 분석 |
|------|-------------|------|
| **컬러** | 다크 배경 + 밝은 텍스트 + 포인트 컬러(보라/파랑 그라데이션) | 개발자 감성, 모던한 인상 |
| **타이포** | 대형 타이틀 + 모노스페이스 코드 폰트 혼용 | 기술 제품 신뢰감 |
| **레이아웃** | 중앙 정렬, 넓은 여백, 섹션 간 명확한 구분 | 콘텐츠에 집중 유도 |
| **인터랙션** | 최소한 (탭 전환, 캐러셀 자동 스크롤 정도) | 로딩 빠르고 직관적 |
| **CTA** | 코드 복사 버튼 (개발 도구 특화) | Vibeollio는 URL 입력/등록 CTA로 변환 필요 |
| **소셜 프루프** | 실제 유저 후기 무한 캐러셀 | 신뢰도 확보에 효과적. Vibeollio도 채택 가능 |

#### Vibeollio에 적용할 수 있는 패턴

1. **싱글 페이지 랜딩** — 메인은 스크롤형 원페이지, 제품 리스트는 별도 페이지
2. **Hero = 핵심 가치 + 즉시 행동** — "제품을 올리세요" CTA를 Hero에 배치
3. **Problem-Solution 포맷** — 제작자의 고민 → Vibeollio의 해결 (bkit과 동일 구조)
4. **Feature Grid 카드** — Vibeollio 핵심 기능을 카드형으로 나열
5. **Social Proof 캐러셀** — 등록된 제품의 피드백/성과 사례
6. **미니멀 네비게이션** — 로고 + 최소 메뉴 (등록, 탐색, 로그인 정도)

#### Vibeollio와 달라야 하는 점

| bkit.ai | Vibeollio |
|---------|-------|
| 개발 도구 → 다크 테마가 자연스러움 | 일반 유저 대상 → 라이트 테마 또는 선택 가능 |
| CTA가 "코드 설치" | CTA가 "URL 등록" 또는 "제품 탐색" |
| 싱글 제품 소개 페이지 | 랜딩 + 제품 피드(목록) 페이지 필요 |
| 리뷰가 보조 역할 | 피드백이 핵심 기능 → 더 눈에 띄게 |

## 6. 추가 기능 (Feature Backlog)

### 6.1 UI 서포터
- 출시 전 UI를 검토/보완해주는 기능
- 디자인 피드백, 접근성 체크 등

### 6.2 Live Overview
- 실제 동작하는 제품의 인터랙티브 프리뷰 제공
- 링크 클릭 없이 플랫폼 내에서 체험

### 6.3 GitHub 보안 검토
- GitHub 레포 링크 입력 시 기본적인 보안 스캔 제공
- 의존성 취약점, 코드 패턴 분석 등

### 6.4 피드백 협업 시스템
- **컨셉:** "올리세요. 솔직하고 실질적인 피드백 드립니다."
- 보안 / 협업 / 보완 / 버그 체크 / 피드백을 통합 제공
- **피드백 기여도** 가시화 (프로필, 랭킹 등)
- **상호보완적 피드백 루프:**
  - 유저가 피드백 작성 → 제작자가 확인/처리
  - 제작자가 "처리 완료" 체크 → 해당 피드백에 배지/표시 부여
  - 피드백 제공자의 기여 이력에 반영

### 6.5 유사 서비스 추천
- 등록된 제품과 비슷한 SaaS/Tool 자동 매칭
- 유저에게는 비교 탐색, 제작자에게는 경쟁 파악

### 6.6 신뢰도 배지
- 랜딩페이지에 삽입 가능한 임베드 배지 제공
- "Vibeollio 인증", 평점, 피드백 수 등 표시
- 예: `<script src="vibeollio.io/badge/product-id">`

- **방향 미정** — 제품 중심 토론? 제작자-유저 직접 소통? 추후 구체화 필요

## 9. 기획 분석 의견

### 강점
- **진입장벽 제로(무료 등록)** — 초기 콘텐츠 확보에 유리
- **피드백 협업 시스템** — 단순 홍보가 아닌 양방향 가치 제공. Product Hunt에 없는 구조
- **신뢰도 배지** — 제작자에게 플랫폼 밖에서도 가치를 줌. 바이럴 효과

### 리스크
- **콜드 스타트** — 제품도 유저도 없는 초기에 어떻게 양쪽을 끌어올 것인가
- **피드백 품질 관리** — 포인트 제도가 양보다 질을 보장하는지 설계 필요
- **"모든 아이디어는 훌륭하다"와 큐레이션의 균형** — 노출 알고리즘 없이는 쓰레기장이 될 수 있음

### 우선순위 제안 (MVP)
1. 제품 등록/조회 (URL + 설명 + 이미지)
2. 피드백 시스템 (작성 + 처리완료 표시)
3. 신뢰도 배지
4. 유사 서비스 추천

나머지(UI 서포터, GitHub 보안 검토, Live Overview 등)는 PMF 검증 후 확장
